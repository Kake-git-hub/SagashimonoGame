# 開発ルール

このファイルは開発時に必ず読み込むこと。

## 必須ルール

### 1. コード変更後は必ずPush
コードを変更・修正した後は、**必ず以下の手順を実行**してください：

```bash
git add -A
git commit -m "適切なコミットメッセージ"
git push
```

- ビルドが成功した場合のみpushする
- コミットメッセージは英語で、変更内容を簡潔に記述

### 2. ビルド確認
変更後に `npm run build` でビルドエラーがないことを確認してからpushする。

## プロジェクト情報

- **リポジトリ**: `Kake-git-hub/SagashimonoGame`
- **デプロイ先**: GitHub Pages (`https://kake-git-hub.github.io/SagashimonoGame/`)
- **技術スタック**: React 19 + TypeScript + Vite

## アプリ概要

「ミッケ」風のもの探しゲームアプリ

### 主要機能
- パズル一覧画面
- ゲーム画面（タブレット横画面/スマホ縦画面対応）
- お題リスト（テキスト/サムネイル切替）
- ヒント機能
- クリア演出（紙吹雪）
- 進捗保存（中断再開対応）
- パズルエディタ

### データ構造
- 座標: 0-1000スケール（画像左上が(0,0)、右下が(1000,1000)）
- 判定半径: 50（5%相当）
- パズルデータ: `public/puzzles/` 配下にJSON形式で配置
